# 横版设计语言适配（万涂幻象三支柱 @1920×1080，祥瑞 2026-06-12 二订）

> **定位纠偏**：这不是"验证过的配方库照搬"——设计永远每片现场创作。
> 本文只回答一件事：design-system.md 的三支柱（编辑排版/Rough 手绘/开源图表）
> 在横版全屏画布上怎么用。**横版的美 = 同一套万涂幻象语言，不是另起炉灶。**

## 去 AI 味禁区（横版同样生效，违者返工）

- ❌ 白卡/玻璃卡居中对称排排坐（横版最容易犯——空间大就想居中堆卡）
- ❌ emoji 当图标/当人物
- ❌ 千篇一律圆角章阵列
- ✅ 编辑排版的不对称、超大编号、细线、留白；手绘的不规则；真实素材的质感

## 支柱一：编辑排版骨架（横版主力）

横版像**杂志跨页**而不是 PPT 单页：
- **左对齐**，禁居中（除 impact 砸字）；视觉重心放在左 1/3 或右 1/3，另一侧留白或放佐证
- **英文 kicker** 眉头（mono 26-30px，大字距，灰或绿）压在大标上方
- **超大编号** 01/02/03（70-90px 粗黑，首条绿）代替图标
- **细线分隔列表**（border-top 1.5px），每条：编号+主词+mono 小标注
- 字号阶梯：大标 80-110 / 条目主词 44-54 / mono 标注 26-34；行高 1.6-2.0
- 竖网格线背景、右上角 mono 角标——编辑页的"版面零件"

## 支柱二：Rough.js 手绘（对治玻璃卡的药）

横版画布大，手绘更出效果：
- 流程/结构/示意一律优先手绘（hachure 填充矩形、抖动箭头、圈注），禁玻璃卡阵
- 手绘 + 编辑排版混排：左编辑列表 + 右手绘图解，是横版最稳的双栏
- demo_js 含 "rough" 自动注入库；静态生成逐帧稳定

## 支柱三：开源图表（ECharts 横向拉满）

- 容器 width 88-92% height 60-66%，animation:false + GSAP clip-path 揭示
- 轴标 ≥28px、数据标签 ≥32px 粗体；图外左上角编辑式标题行（大标+mono 副标）

## 横版专有件（非模板，是画布约定）

- **章节分段条**：顶部 58px，品牌格嵌最左，全程导航（render 自动）
- **画面叠字 overlay**：媒体镜标注画面身份（title ≤10 字+sub 来源），治"画面口播不匹配"
- **人物镜**：照片+左对齐身份列（kicker 日期+大标动作+mono 履历），历史人物灰调
- **图源小签**：22px mono rgba(255,255,255,.45) 角落；**截图里出现二维码必须打码**
  （cv2 QRCodeDetector 检测+高斯模糊，详见 asset-sourcing）

## 硬底线（每镜自检）

1. 产品/公司名出现 → 官方 logo 或界面截图同屏（白底章只包 logo 本体，不当版式语言）
2. 每屏有主视觉锚点（编号列表/手绘图/大数字/照片/图表之一），纯文字铺排=返工
3. 左对齐编辑感 ≥ 居中；不对称 ≥ 对称
4. 媒体镜带 overlay 或 source；二维码打码
