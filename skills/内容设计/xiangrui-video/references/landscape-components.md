# 横版设计语言适配（万涂幻象三支柱 @1920×1080，祥瑞 2026-06-12 二订）

> **定位纠偏**：这不是"验证过的配方库照搬"——设计永远每片现场创作。
> 本文只回答一件事：design-system.md 的三支柱（编辑排版/Rough 手绘/开源图表）
> 在横版全屏画布上怎么用。**横版的美 = 同一套万涂幻象语言，不是另起炉灶。**

## 去 AI 味禁区（横版同样生效，违者返工）

- ❌ 白卡/玻璃卡居中对称排排坐（横版最容易犯——空间大就想居中堆卡）
- ❌ emoji 当图标/当人物
- ❌ 千篇一律圆角章阵列
- ✅ 编辑排版的不对称、超大编号、细线、留白；手绘的不规则；真实素材的质感

## 支柱一：编辑排版骨架（横版主力）

横版像**杂志跨页**而不是 PPT 单页：
- **左对齐**，禁居中（除 impact 砸字）；视觉重心放在左 1/3 或右 1/3，另一侧留白或放佐证
- **英文 kicker** 眉头（mono 26-30px，大字距，灰或绿）压在大标上方
- **超大编号** 01/02/03（70-90px 粗黑，首条绿）代替图标
- **细线分隔列表**（border-top 1.5px），每条：编号+主词+mono 小标注
- 字号阶梯：大标 80-110 / 条目主词 44-54 / mono 标注 26-34；行高 1.6-2.0
- 竖网格线背景、右上角 mono 角标——编辑页的"版面零件"

## 支柱二：Rough.js 手绘（对治玻璃卡的药）

横版画布大，手绘更出效果：
- 流程/结构/示意一律优先手绘（hachure 填充矩形、抖动箭头、圈注），禁玻璃卡阵
- 手绘 + 编辑排版混排：左编辑列表 + 右手绘图解，是横版最稳的双栏
- demo_js 含 "rough" 自动注入库；静态生成逐帧稳定

## 支柱三：开源图表（ECharts 横向拉满）

- 容器 width 88-92% height 60-66%，animation:false + GSAP clip-path 揭示
- 轴标 ≥28px、数据标签 ≥32px 粗体；图外左上角编辑式标题行（大标+mono 副标）

## 横版专有件（非模板，是画布约定）

- **章节分段条**：顶部 58px，品牌格嵌最左，全程导航（render 自动）
- **画面叠字 overlay**：媒体镜标注画面身份（title ≤10 字+sub 来源），治"画面口播不匹配"
- **人物镜**：照片+左对齐身份列（kicker 日期+大标动作+mono 履历），历史人物灰调
- **图源小签**：22px mono rgba(255,255,255,.45) 角落；**截图里出现二维码必须打码**
  （cv2 QRCodeDetector 检测+高斯模糊，详见 asset-sourcing）

## 版式骨架五件套（2026-06-13 拆解抖音知识区头部账号内化，VOL.10 实测）

> 来源：拆解抖音 PPT 动画类知识账号（17 万粉级）三条爆款逐帧归纳，嫁接到万涂幻象语言上。
> **定位：版式骨架是"讲法"不是"模板"**——每件的内容、文案、节奏仍然每片现场设计，
> 连续两片用同一件套同一长相 = 模板化返工。五件按叙事需要选用，没有必选项。

1. **巨字情绪卡**——讲到情绪节点/转折/吐槽爆点，整屏大字砸出来（200-260px，
   YSBTH 标题字，白+粉莓/翠绿，轻微 rotate(-2~-3deg)，同色 text-shadow 发光）。
   作用：给密集干货打情绪标点。一片 1-2 处，多了廉价。
2. **路标进度卡**——长片(>3 分钟)或多步骤内容，"先 X → 再 Y → 后 Z"三卡横排，
   GSAP 跟口播节拍逐卡点亮（当前卡绿渐变高亮+发光，其余暗置）。
   作用：观众永远知道讲到哪了，治长视频中途流失。顶部章节条是被动导航，这个是主动路标。
3. **图标流程卡**——概念关系/数据流向，用 Rough.js 手绘节点（人/机器/箱子/文档）+
   抖动箭头 + 箭头上标短语（"一句话"/"全自动"）。比文字列表直观，比真截图轻。
4. **实操卡**——教程/安装/命令类内容三件套：白粗边框截图或仿终端（border 3px 白 + 圆角 12 +
   深投影）+ 粉莓批注框（圈住关键行 + 角标签"就发这一句"）+ 右侧数字步骤栏
   （绿圆数字当前步 + 灰圆其余步）。评论区提问密度的来源，照着就能做。
5. **常驻角色卡**——手绘漫画角色出场陪观众（生成与抠图管线见 asset-sourcing.md 第 7 节）。
   作用：给产线一个"人"，情绪由角色表演（熬夜弓背/兴奋举手），口播只管信息。
   角色是品牌资产可跨片复用（同一形象不同姿势），姿势按当片剧情现生成。

**字体纪律（2026-06-13 定，VOL.10 翻车换来的）**：
- 大字/标题只用品牌标题字 `font-family:YSBTH`（@font-face 已注册，实体是随 skill 附带的
  AlibabaPuHuiTi-Heavy）——**禁止 Baoli SC / 隶变 / 楷体等系统花字体**，圆软笔触和编辑骨架严重不搭
- demo_html 的 style 属性值里**禁止再出现单引号**（外层属性就是单引号包的，
  `style='font-family:'YSBTH''` 会把属性截断、字号全塌）。字体名不带引号写：
  `font-family:YSBTH,sans-serif`（无引号的含空格字体名 CSS 同样合法：`PingFang SC` 直接写）

## 硬底线（每镜自检）

1. 产品/公司名出现 → 官方 logo 或界面截图同屏（白底章只包 logo 本体，不当版式语言）
2. 每屏有**图像级**主视觉锚点（真实素材/手绘图/画板风成图/图表/角色/截图/logo），
   且与当句口播精准对应；编号列表、大数字这类文字型版面不算锚点，只能当节奏点缀
   （全片 ≤2 镜），纯文字铺排=返工（祥瑞 2026-06-13 收紧：图主文辅）
3. 左对齐编辑感 ≥ 居中；不对称 ≥ 对称
4. 媒体镜带 overlay 或 source；二维码打码
5. 绝对定位的文字标签**必加 white-space:nowrap**（absolute+inline 会收缩成竖排字，
   2026-06-12 印章镜实锤）；文档/截图素材一律 `"fit":"card"` 卡装不裸铺
