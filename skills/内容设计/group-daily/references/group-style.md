# 群风格指纹 · 长期沉淀资产

这是 group-daily skill v2 的核心升级。**只沉淀群文化，不沉淀个人画像**——因为群文化稳定（内部黑话、典型徽章、禁忌长期不变），而人是动态的（不固化，每次临时分析）。

## 一、为什么只沉淀群风格

第一版 skill 每次都从零判断群文化：AI 通读消息 → 凭直觉判断 → 写出来。

问题在于：

- 同一个群第二天再写，调性可能跟前一天不一样
- “内部黑话”“自创词”一旦没被记下，第二天 AI 容易改造成普通话（“示例黑话B”被翻译成“群友群体”就毁了）
- 已经踩过的“禁忌”（比如不要砍 示例联系人C 的诊断细节、不要丢示例联系人D的概念归功）每次都要重新发现

群风格沉淀的价值是**让群的内部黑话、调性、禁忌不流失**。

**为什么不沉淀个人画像**：

- 人是动态的，今天的金句担当明天可能是提问者
- 一旦固化“他是 X”，AI 写下次故事会强迫他扮演 X
- AI 临时分析当天发言的人物气质本来就够准（v1 三份日报实测）
- 写人物档案有隐私感，长期沉淀“研究”群友不舒服

## 二、Vault 目录结构


styles 是 markdown 文件，AI 用 Read / Write / Edit 直接操作。

## 三、群风格指纹 `styles/<群名>.md`

### 模板

```markdown
---
group_name: 示例群
created: 2026-05-12
last_updated: 2026-05-12
sample_dates: [2026-05-11]
sample_count: 1
---

# 示例群 · 群风格指纹

## 一句话定位
群主周围的 AI 文艺青年聚集地，文学性、段子密度、社会实验意味同时拉满。

## 主流话题
- AI 个人 agent 拟人化（Alice / Cola）
- 示例主流话题A
- skill 创作（pua skill / 巴别渊 skill）

## 文化标签
玩梗 / 文艺 / 社会实验 / 互助 / 段子

## 典型徽章库
段子手 / 群史官 / 召集人 / 神金句 / 示例黑话B / 反优化哲学家 / 旁观者 / 蹦迪者

## 调性
- 衬线感强，群主爱写《群本纪》体文言文小记
- 玩梗密度高，自创词随时出现
- 长金句 + 一字短回应交替

## 内部黑话
- 示例话题 / 示例梗A / 分白
- 示例黑话B（Alice / 艾莉周边玩梗群体）
- 示例黑话C
- 群本纪 / Ka21·群本纪

## 禁忌
- 不要写成“高效会议纪要”式扁平总结
- 不要砍掉文学性
- 大事件如“Alice 被定性为 AI 社会实验”必须保留原话
- 不要 emoji 堆砌

## 角色类型（描述群的人才结构，非点名）
这个群里通常有这些类型的活跃者：召集人 / 文案高手 / 段子王 / 工作流玩家 / 旁观吃瓜派 / 示例黑话B成员。

具体每天谁扮演哪个角色，由 AI 当天临时判断，不在这里点名固化。
```

### 字段说明

| 字段 | 说明 |
|---|---|
| `group_name` | 与 wechat 群名完全一致 |
| `created` | 首次生成日期 |
| `last_updated` | 最近一次更新日期 |
| `sample_dates` | 用过哪些天的数据来归纳风格 |
| `sample_count` | 已积累的样本日报数 |
| 一句话定位 | 用一句话讲清楚这个群是干什么的 |
| 主流话题 | 4-8 个长期反复出现的话题 |
| 文化标签 | 3-7 个气质关键词 |
| 典型徽章库 | 这个群常用的角色徽章（不绑定具体人） |
| 调性 | 3-5 条写作风格指引 |
| 内部黑话 | 群里的自创词、典故、梗 |
| 禁忌 | 不要怎么写 |
| 角色类型 | 描述这个群有哪些类型的角色（**不点名固化**，留给 AI 当天判断） |

## 四、加载和更新流程

### 跑日报前加载（SKILL.md Step 2.5）

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/context_helper.py check-style \
    --group-name "<群名>"
```

- 如果 `exists: true`，用 Read 加载 `styles/<群名>.md`。把里面的一句话定位、文化标签、典型徽章库、内部黑话、禁忌作为写故事的硬约束。
- 如果 `exists: false`，跳过加载（首次跑某群），Step 7.5 会生成 v1 风格。

**人物画像不预加载档案**，AI 当天读聊天记录临时分析。

### 写完日报后更新（SKILL.md Step 7.5）

```bash
python3 ${CLAUDE_SKILL_DIR}/scripts/context_helper.py path \
    --style "<群名>"
```

- 已存在：用 Edit 推进 `last_updated`、追加 `sample_dates`、`sample_count + 1`。如果今天出现了新的内部黑话、新的徽章类型、新的禁忌观察，加入对应小节。
- 不存在：用 Write 生成 v1，按上面的模板填。frontmatter 里 `created` 和 `last_updated` 都填今天，`sample_dates` 填 `[今天]`，`sample_count` 填 1。

**人物相关的观察不沉淀到任何持久档案**，留在当天的 story.json 里即可。

## 五、AI 写作时如何使用群风格

加载完 style 后，写故事环节遵循：

- “一句话定位”决定开场叙述的方向
- “文化标签”决定 lead_title 和故事正文的语气
- “典型徽章库”优先复用（保持一致性），但允许当天创造新徽章
- “内部黑话”必须保留并自然嵌入（不能改造成普通话）
- “禁忌”是硬约束，写完检查每一条
- “角色类型”是参考清单，不强制套到具体人头上

## 六、继承与突破

- 默认遵循已有 style
- 但今天破例的部分（如群里突然进入新话题、出现新黑话）要在 Step 7.5 更新档案时记下来
- 继承让日报保持一致性，突破让档案不断进化
