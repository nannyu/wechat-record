# 群日报设计准则

> 来自《群日报形态横纵分析报告》（2026-05-11，存在 Vault `30.资源/研究报告/`）的核心结论。在生成任何一份群日报前应该把这些准则当作硬约束。

## 一、范式定位：故事化年报，不是智能纪要

群日报存在的意义是把社群一天的对话，转化成可消费、可传播、可沉淀的内容产品。它不是飞书/钉钉那种 AI 自动生成的“会议纪要”。

智能纪要的范式是“保存信息”，故事化年报的范式是“传递价值”。前者解决“文档化”，后者解决“人想看、想分享、想记住”。

把每份群日报当作一篇短篇报道写，不是一份会议总结。

## 二、时间故事线是唯一骨架

QFD 质量屋矩阵给出的优先级排序：

| 技术特征 | 绝对权重 | 含义 |
|---|---|---|
| **时间故事线** | **200** | 整份日报的骨架，所有内容都挂在时间锚点上 |
| **话题聚类** | **198** | 每个时间锚点归属一个主题/角色 |
| AI 摘要核心观点 | 191 | 每段配 1-3 句金句引用 |
| 卡片设计可分享 | 162 | 视觉上每段可独立截图 |
| 搜索功能 | 54 | 砍掉 |

**所有不在这条骨架上的模块都是冗余**，特别是：

- ❌ 24h 活跃柱状图
- ❌ 话题分布横向条形
- ❌ TOP N 输出者排行
- ❌ 互动数据多宫格
- ❌ 全员人物标签册（46 张那种）
- ❌ Dashboard / KPI 磁贴
- ❌ Sticky 导航条

留下：

- ✅ Masthead 报头 + Lead 主标题 + 开场叙事
- ✅ 时间故事线 6-8 个节点
- ✅ 今日高光人物 6-8 张
- ✅ 折叠干货（SOP + Q&A，默认 details open）
- ✅ Colophon 底部 5 个核心数字

## 三、五个本质转移（来自纵向分析）

百年类日报演化的核心转移，做群日报时要往后面这一列走：

| 旧范式 | → | 新范式 |
|---|---|---|
| 信息权力 | → | **关系信任** |
| 被动消费 | → | **主动认同** |
| 信息速度 | → | **意义创造** |
| 广泛覆盖 | → | **深度聚集** |
| 现在发生了什么 | → | **你的故事是什么** |

具体到一份群日报：
- 不写“群里发了多少条消息”，写“群里今天有人提了一个问题，引发了什么”
- 不堆 46 个全员标签，放大 6-8 个最有戏的人物
- 不追求全面覆盖，追求情节连贯

## 四、视觉去 AI 化（杂志风）

群日报第四版被群友批评“太 AI 了”。第六版做了杂志风重构，要保持下去。

**字体**：
- 中文用衬线字体（Songti SC / STSong / Noto Serif CJK SC）
- 英文用 Georgia
- 拒绝系统默认 sans-serif（SF Pro / PingFang）

**配色**：
- 底色：米黄纸色 #ede5d0
- 主文字：墨黑 #1a1815
- 高亮色：朱砂红 #b8443c
- 次要色：橘金 #d4a017
- 灰褐：#8a8270
- 拒绝绿色商务渐变、蓝色 SaaS 风

**排版**：
- 大编号“01-08”用 Georgia 衬线超大字（96px 朱砂红）
- 首字下沉（drop cap）大红色衬线
- 横线代替卡片边框（border-top: 4px solid + border-bottom: 1px solid 的 Masthead）
- 大留白
- 拒绝 emoji 堆砌（最多保留产出物前的 📦 或彻底不用）

**装饰**：
- 引用块左侧 3px 朱砂红竖线 + 大衬线斜体 + 中文双引号“”
- 头像 + 4px 4px 0 朱砂红偏移阴影（杂志感投影）
- 大写英文小字做副标签（CAST / PRODUCED / Vol. / Stories）

## 五、写作风格底线

**严禁**：

- AI 套话：“首先”“综上所述”“值得注意的是”“赋能”“打造闭环”
- 破折号 ——
- emoji 满天飞
- 流水账：“09:20 示例联系人A问了问题。11:18 万重山追问”

**鼓励**：

- 故事化叙事：有主角、有冲突、有解决
- 事实后追评：引用完一句金句，加一句自己的判断
- 节奏感：长短句交错，有时候一句话独立成段
- 中文双引号“”+ 朱砂红装饰
- 口语化但不失重量：可以说“这玩意儿”“炸开了”，但不能写流水账

## 六、每个时间节点的固定结构

```
no:        "01" - "08"
time:      "09:20"
badge:     "提问者" / "架构师" / "救火队员" 等角色徽章（不是分类标签）
cast:      [{ name, wxid }]  主角们，配真实头像
theme:     大标题，一句话故事概括
story:     150-250 字叙述，衬线大字两端对齐
quotes:    1-3 句金句引用，配作者
output:    一行 PRODUCED 标签 + 产出物
```

不是每段都要 8 个节点。少则 6 个，多则 8 个，看那天群里的实际戏剧密度。

## 七、保留干货折叠为 details open

SOP 和 Q&A 用 `<details open>` 而不是 `<details>`，因为：

- PNG 长图截图时折叠区只会截到 summary 那行
- 群友打开 HTML 看时仍可以手动收起
- 视觉上保持“可折叠”的杂志层级感

这是踩过的坑，写死在 render_html.py 里了。
