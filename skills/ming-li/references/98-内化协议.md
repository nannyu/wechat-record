# 新书内化协议（Schema 维护 SOP）

> 当示例命主说"内化《XX 书》" / "学一下《XX》" / "把这本书变成你的知识" / 给一个 PDF → 走本 SOP。
> **目标**：让命理大师 schema 随新经典自动升级，避免知识碎片化。

## 一、内化触发

用户说：
- "内化《XX 书》"
- "学一下《XX》"
- "把这本书变成你的知识"
- 提供 PDF 路径或电子书链接
- "我要学梅花易数 / 奇门 / 风水 ..."（开新模块）

## 二、内化 SOP（10 步）

### 步骤 1：资源获取

**优先级**：
1. **文本层 PDF**（直接可读，最优）
2. **EPUB / TXT**（次优）
3. **在线全文**（ctext.org / 维基文库 / shuge.org）
4. **扫描 PDF + 清晰**（多模态视觉读）
5. **扫描 PDF + 模糊**（OCR 或弃用）

### 步骤 2：保存原书到 Vault

```bash
Vault 20.领域/命理学/[原书名]-[版本说明].pdf
```

例：
- `命运的求索-陆致极.pdf`
- `滴天髓.pdf`
- `紫微斗数全书-含文本层.pdf`

### 步骤 3：阅读

- **小书 < 30 页**：主对话直接 Read 全文
- **中书 30-100 页**：选关键章节 Read
- **大书 100+ 页**：定核心章节 → Read + researcher 协助
- **PDF > 32MB**：先用 pymupdf 切分

### 步骤 4：写化文档到 Vault

```bash
Vault 20.领域/命理学/[书名]内化-[作者].md
```

**模板**：
```markdown
---
title: [书名] 内化（[作者]）
date: YYYY-MM-DD
tags:
  - 领域/命理学
  - 内化
  - [书名]
source: [书的全名] · [作者] · [版本] · [页数]
status: [框架完成 / 全量完成 / 待校]
type: [方法论新域 / 神煞补遗 / 格局补遗 / etc.]
---

# [书名] 内化 · [作者]

## 一、本书的地位与方法论
## 二、核心理论
## 三、独有贡献（vs 其他经典）
## 四、应用于示例八字的判断
## 五、对 schema 的升级建议
```

### 步骤 5：抽取入 schema（关键 ⭐）

**根据内容性质，自动同步到对应 references/**：

| 内容类型 | 写入到 |
|---|---|
| **新神煞** | 追加 `11-八字-神煞库.md` 对应类别 |
| **新格局** | 追加 `13-八字-格局学.md` 对应章节 |
| **新十神组合** | 追加 `12-八字-十神组合.md` |
| **十干十二支新解** | 追加 `01-排盘基础.md` |
| **大运/流年新法** | 追加 `90-应期算法.md` |
| **紫微主星补遗** | 追加 `20-紫微推演.md` |
| **紫微格局补遗** | 追加 `20-紫微推演.md` |
| **六爻新案例** | 追加 `30-六爻推演.md` |
| **六爻特殊技法** | 追加 `30-六爻推演.md` |
| **新方法论体系**（如风水）| **新建 references/40-风水模块.md** |
| **新经典与四家对比** | 追加 `14-八字-四家对照.md` |
| **命主专属新判断** | 追加 `91-命主专属档案.md` |

### 步骤 6：更新升级日志

写到 `references/99-升级日志.md`：

```markdown
## YYYY-MM-DD 内化《XX 书》(作者)

**入库文件**：Vault 20.领域/命理学/XX内化-作者.md
**升级影响**：
- references/11 新增 [N] 个神煞
- references/13 新增 [M] 个格局
- references/91 示例命主新发现 [...]
```

### 步骤 7：更新顶层 SKILL.md

如有重大变化：
- 已内化经典清单（§ 四）
- 触发词（如新增"奇门遁甲"则更新触发表）
- 知识 Schema 当前文件清单（§ 二）

### 步骤 8：自检

跑完整流程后回头看：
1. 化文档是否完整？（不是只有提纲）
2. references/ 是否真正同步了关键内容？（不是只在化文档里写）
3. 升级日志是否记录？
4. 顶层 SKILL.md 是否更新清单？

任一答 "否" → 回去补。

### 步骤 9：命主专属应用

如果新经典对示例八字有新判断：
- 追加到 `91-命主专属档案.md`
- 标注"基于 XX 经典 / XX 作者"

### 步骤 10：通知示例命主 + pbcopy 摘要

输出给示例命主一个 < 200 字摘要：
- 本次内化什么书
- 核心新发现 N 个
- schema 哪几个文件被升级
- 示例八字新判断（如有）

## 三、新开模块的规则

### 编号分配

| 编号区间 | 类别 |
|---|---|
| 40-49 | 风水（阳宅/阴宅）|
| 50-59 | 梅花易数 |
| 60-69 | 奇门遁甲 |
| 70-79 | 大六壬 |
| 80-89 | 相术（手面相）|

### 新模块开模 SOP

新模块第一本经典内化时：

1. 建顶层模块文件 `references/40-风水模块.md`（例）
2. 顶层文件结构：
   ```
   # 风水模块（阳宅 / 阴宅）
   ## 一、本模块的方法论分类
   ## 二、核心理论
   ## 三、操作步骤 SOP
   ## 四、与其他模块的协作
   ## 五、纪律与红线
   ```
3. 若该模块多本经典 → 拆分子文件 `41-风水-阳宅.md` / `42-风水-阴宅.md`
4. 更新顶层 SKILL.md 的"触发词"+"知识 Schema"+"路由表"

## 四、内化质量检查清单

新书内化后，确认以下：

- [ ] Vault 内化文档 ≥ 5000 字
- [ ] 至少 3 个章节（理论 / 应用 / 对比）
- [ ] 对示例八字有具体判断（≥ 1 条新发现）
- [ ] schema references/ 至少 1 个文件被升级
- [ ] 99-升级日志.md 有记录
- [ ] 顶层 SKILL.md 经典清单更新
- [ ] 命例库索引（如有）更新

## 五、内化禁忌

### ❌ 不该做的

1. **不要照搬古书**——必须现代化转述
2. **不要瞎编**——不确定的明确标"待考"
3. **不要堆砌信息**——挑核心，不要 200+ 神煞全照搬
4. **不要孤立内化**——必须与已有四家对比
5. **不要跳过命主专属判**——每本书都要给示例八字一个交代

### ✅ 该做的

1. **诚实标注信息源**（页码 / 章节）
2. **对比现有经典**（这本书有什么新东西）
3. **白话翻译古文**
4. **结论可执行**（不是空理论）
5. **schema 真正升级**（不是只在化文档里改）

## 六、内化失败的处理

如果内化失败（PDF 不可读 / 资源找不到 / 内容过老 / 与四家冲突太多）：

1. **明确写明失败原因**到化文档
2. **不强行内化**——宁缺勿乱
3. **建议替代**：如紫微古书读不了 → 用现代整理本
4. **回报示例命主**：告诉哪步失败，需要什么帮助

## 七、批量内化（多本书并行）

如果示例命主一次给多本书：

1. **分别评估**每本书的可行性
2. **优先级排序**（重要程度 / 资源完整度）
3. **顺序内化**——不并行（避免互相影响 schema）
4. **每本完成后立刻 self-check**
5. **全部完成后**给示例命主一个总报告
