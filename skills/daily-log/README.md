# daily-log · 收工日志

> 一句收工，把你今天在飞书的全链路足迹自动重建成一份带链接、能 @ 人的工作枢纽文档。
> 不是流水账，是当天工作的索引中枢。

一个 Claude Code Skill，配合飞书官方开源的 **lark-cli**，把你一天在飞书里的消息、会议、妙记、文档、任务、审批、OKR、考勤、邮件全自动聚合，按第一人称整理去重、解析对接人实名与公司，生成一份结构化飞书文档：速览数字卡、工作全景思维导图、一天时间线、关键产出、会议三链接、沟通闭环、文档价值分级、事项明细、待办。

每个人名是真 @、每个会议挂着妙记 / 智能纪要 / 文字记录三链接、每篇文档能点开、每个群聊给入口，日志里的实体都能一键跳转。

---

## ✦ 为什么做这个

市面上的收工日报要么靠人手填 回忆半天，写成流水账，要么是会议智能纪要 信息保存了，但还是平铺。

这个 skill 走另一条路：**数据不靠回忆，靠飞书自动扒；价值不靠平铺，靠结构化加判断。**

设计依据基于一份《AI 收工日志 skill 横纵分析报告》，纵向梳理了日报这个品类七十年的演进 从工厂交接本到 AI 自动聚合，横向对比了飞书 / 钉钉日报、Slack standup 机器人、Git 自动日报等阵营。核心判断是：

> AI 自动日报真正的价值，不是让交差更省力，而是借自动聚合，让给老板看第一次不需要表演，因为数据是真的，表演不了。

---

## ✦ 飞书数据是怎么来的 重点

全部基于飞书官方今年 3 月开源的命令行工具 **lark-cli** `@larksuite/cli`，专为 AI Agent 设计，GitHub 已破万星。本 skill 只调用它的**只读**能力，在你**自己的飞书账号、自己有合法访问权的数据**上操作：

- 消息、日程、会议妙记、文档、任务、审批、OKR、考勤、邮件，覆盖飞书 19 个业务域
- 对接人实名解析走 `contact +search-user --user-ids ... --as user` 实名在 `localized_name`
- 生成文档走 `docs +create --api-version v2`，画板走 `whiteboard +update`

完整命令、字段、踩过的八条坑，全在 [`references/lark-recipes.md`](references/lark-recipes.md)。

## ✦ 安装 / 用法

前提：装好飞书官方 [lark-cli](https://github.com/larksuite/cli) 并完成授权 `lark-cli doctor` 返回 `"ok": true`。

把 `daily-log/` 整个目录放进 Claude Code 的 skills 目录，对 AI 说一句收工今天干了啥复盘一下今天即可。skill 会自动跑完采集 → 整理 → 生成飞书文档的全流程。
